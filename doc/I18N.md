# Gestione Lingue

Le lingue sono gestite tramite file JSON.

In sviluppo i file si trovano in:

```text
backend/locales/
```

Nell'app portable vengono copiati in:

```text
PlannerApp/locales/
```

## File lingua

Esempio:

```text
it.json
en.json
```

Il nome del file corrisponde al codice lingua.

Esempi validi:

```text
it
en
fr
es
pt-BR
```

## Struttura JSON

Il progetto usa un dizionario piatto:

```json
{
  "settings.title": "Impostazioni",
  "sideBar.project": "Progetti",
  "taskModal.actions.save": "Salva"
}
```

Questa struttura e semplice da leggere nel frontend:

```jsx
jsonLanguage['settings.title']
```

## Aggiungere una nuova lingua

Per aggiungere una lingua nell'app portable:

1. copia un file esistente, per esempio `it.json`;
2. rinominalo, per esempio `fr.json`;
3. traduci i valori;
4. riavvia o ricarica l'app.

Il backend espone le lingue disponibili leggendo i file presenti in `LOCALES_DIR`.

## Regola importante

I file esterni in `PlannerApp/locales/` non vengono sovrascritti all'avvio.

Se manca un file default, viene copiato dal bundle. Se invece esiste gia, viene lasciato invariato per non perdere modifiche dell'utente.
